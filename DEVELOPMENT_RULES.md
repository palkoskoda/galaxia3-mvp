# 📋 Pravidlá vývoja - Galaxia3

> **Golden Rule:** *Ak to neprejde lokálnym buildom, NEpushuj to na GitHub.*

---

## 🚫 Čo ZAKAZUJEME (aby sme sa vyhli chybám)

### 1. ZAKAZANÉ: Push bez lokálneho testu
```
❌ git add . && git commit && git push
✅ ./build-check.ps1 && git push
```

### 2. ZAKAZANÉ: Používať `any` bez komentára
```typescript
❌ const data: any = fetchData()
✅ const data: UserData = fetchData() // alebo
✅ const data = fetchData() as UserData // s komentárom prečo
```

### 3. ZAKAZANÉ: Meniť verzie balíkov bez dôvodu
```json
❌ "express-rate-limit": "^7.1.5"  // ^ povolí nové verzie
✅ "express-rate-limit": "6.10.0"  // Presná verzia
```

### 4. ZAKAZANÉ: Ignorovať TypeScript chyby
```typescript
❌ // @ts-ignore
✅ // @ts-expect-error - vysvetlenie prečo to dočasne ignorujeme
```

### 5. ZAKAZANÉ: Mixovať package manažérov
```
❌ npm install lokálne + yarn na Render
✅ Používame iba npm všade
```

## ✅ Povinný workflow (krok za krokom)

### Pred každým pushom:

```powershell
# Krok 1: Spusti build check
.\build-check.ps1

# Krok 2: Ak prejde, pushni
if ($LASTEXITCODE -eq 0) {
    git add .
    git commit -m "feat: popis zmien"
    git push origin master
} else {
    Write-Host "❌ Oprav chyby pred pushom!"
}
```

### Čo build-check.ps1 overuje:
1. ✅ Čistá inštalácia (`npm ci`)
2. ✅ TypeScript kompilácia (`tsc`)
3. ✅ Build frontendu (`vite build`)
4. ✅ Existencia výstupných súborov (`dist/`)

## 🔧 Technické štandardy

### TypeScript:
```json
{
  "compilerOptions": {
    "strict": true,           // VŽDY zapnuté
    "noImplicitAny": true,    // Zakážeme any
    "strictNullChecks": true, // Null kontrola
    "skipLibCheck": true      // Iba pre externé knižnice
  }
}
```

### Package.json:
```json
{
  "engines": {
    "node": "20.11.0"  // Presná verzia
  },
  "dependencies": {
    "express": "4.18.2"  // BEZ ^, presná verzia
  }
}
```

### Environment:
- **Lokálne:** `.env` súbor (gitignored)
- **Produkcia:** Environment variables v Render/Railway
- **Nikdy:** Necommitovať secrets do repa

## 🐛 Ako riešiť chyby (postup)

### Chyba: `Type 'X' is not assignable to type 'Y'`
1. Skontroluj importy - správny súbor?
2. Skontroluj interface/type definíciu
3. Pridaj explicitný type cast ak je to legitímne
4. NEpoužívaj `any` - príčina problému

### Chyba: `Module not found`
1. `npm install` v správnom adresári
2. Skontroluj `package.json` či je balík v dependencies
3. Skontroluj cestu v importe

### Chyba: Build zlyháva na Render ale nie lokálne
1. Skontroluj `engines` v package.json
2. Skontroluj `.node-version`
3. Použi `npm ci` namiesto `npm install`
4. Skontroluj environment variables

## 🎯 Konkrétne pravidlá pre tento projekt

### Rate Limiting:
```typescript
// ✅ Správne - priamo v app.use
app.use('/api/', rateLimit({...}))

// ❌ Zlé - ukladanie do premennej
const limiter = rateLimit({...})
app.use('/api/', limiter)  // TypeScript problém
```

### Database:
```typescript
// ✅ Správne - SQLite cesta z env
const DB_PATH = process.env.DATABASE_URL || './galaxia3.db'

// ❌ Zlé - hardcoded cesta
const DB_PATH = './galaxia3.db'
```

### Error Handling:
```typescript
// ✅ Správne - custom error type
interface ErrorWithCode extends Error {
  code?: string
}
const err = error as ErrorWithCode

// ❌ Zlé - direct access
error.code  // TypeScript error
```

## 🚨 Čo robiť keď niečo zlyhá

### Build zlyháva:
1. Čítaj chybu odhora nadol
2. Prvá chyba je tá podstatná
3. Oprav ju
4. Spusti build znova

### Aplikácia nebeží na Render:
1. Skontroluj logy na Render Dashboard
2. Skontroluj environment variables
3. Skontroluj či prešiel GitHub Actions
4. Ak Actions prešiel, problém je na Render (settings)

### TypeScript sa sťažuje na nový kód:
1. Pridaj správny typ
2. Ak nevieš typ, nájdi ho v existujúcom kóde
3. NEpoužívaj `any` ako prvú možnosť
